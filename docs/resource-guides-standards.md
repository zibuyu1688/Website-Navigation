# Resource Guides 收录标准

本文件用于统一 Resource Guides 的二级分类口径。

总原则：

- 资源导引优先收录“帮助用户发现候选”的网址，而不是“让用户直接执行任务”的网址。
- 如果一个站点的第一价值是帮助用户筛选工具、进入生态、观察趋势、获取信息源，应优先归入 Resource Guides。
- 如果一个站点的第一价值是直接使用、直接接入、直接部署、直接照做，应优先归入具体业务分类或 Technical Playbooks。

## 二级分类

### 1. 工具导航

定义：目录站、聚合站、找工具入口。

收录：

- AI 工具目录
- 任务型工具检索站
- 聚合式产品发现平台
- 多工具对比入口

不收录：

- 单一工具官网
- 平台后台
- 官方 API 文档
- 具体教程页

判断问题：

- 用户来这里，是不是先找候选工具，而不是立刻开工。

### 2. 模型与社区

定义：模型生态、数据集社区、开源社区、技能市场。

收录：

- 模型社区
- 数据集平台
- 开源生态入口
- Agent 技能市场
- 开发者社区与资源生态

不收录：

- 单篇教程
- 单篇新闻
- 纯工具目录站
- 单一 SaaS 官网

判断问题：

- 用户来这里，是不是为了进入某个模型或开发者生态，而不是消费一篇内容或直接使用单点工具。

### 3. 产品研究

定义：产品库、竞品、趋势、赛道、需求验证。

收录：

- 产品数据库
- 竞品观察平台
- 趋势发现站
- 赛道观察与市场研究入口
- 需求验证工具

不收录：

- API 文档
- 部署文档
- 分步骤教程
- 工程复盘文章

判断问题：

- 用户来这里，是不是为了判断做什么、选什么、看谁在增长。

### 4. 学习与资讯

定义：新闻、周报、解读、知识整理、非步骤型学习。

收录：

- 行业资讯站
- 周报与快讯
- 深度解读
- 专题观察
- 知识整理与内容策展
- 非操作型学习材料

不收录：

- 明确的实操教程
- 代码 walkthrough
- 部署指南
- 官方文档

判断问题：

- 用户来这里，是不是以阅读、理解、跟踪信息为主，而不是照着操作。

## 与 Technical Playbooks 的边界

Resource Guides 回答：

- 去哪里找资源
- 去哪里找候选工具
- 去哪里看生态、趋势和信息源

Technical Playbooks 回答：

- 具体怎么做
- 怎么接入
- 怎么部署
- 别人怎么做成的

因此以下内容不优先进入 Resource Guides：

- 教程实战
- 自动化工作流
- 部署与运维
- 案例拆解

## 归类优先级

当一个网址看起来同时符合多个分类时，按“第一价值”判断：

1. 先判断它是不是帮助用户发现候选。
2. 如果是，再判断它主要是在找工具、进生态、看产品趋势，还是看信息内容。
3. 如果不是帮助用户发现候选，而是帮助用户直接执行，就不要放 Resource Guides。
